## 📖 Lecture — Java OOP Topics

> The lecture covers all 13 topics below. Each is explained with a clear definition, key concepts, rules, and real-world context.

<details>
<summary><b>☕ 01 — Java Overview</b></summary>
<br/>

An introduction to the Java programming language — what it is, how it works, and the environment it runs in. Covers the difference between **JVM**, **JRE**, and **JDK**, and the compilation process from source code to execution.

![JVM](https://img.shields.io/badge/JVM-Virtual%20Machine-8B0000?style=flat-square)
![JRE](https://img.shields.io/badge/JRE-Runtime%20Environment-800000?style=flat-square)
![JDK](https://img.shields.io/badge/JDK-Development%20Kit-darkred?style=flat-square)

</details>

---

<details>
<summary><b>🧱 02 — Class and Object</b></summary>
<br/>

The foundational building blocks of OOP. Explains what a **class** is as a blueprint and what an **object** is as an instance. Covers fields, methods, constructors (default, parameterized), and the `this` keyword.

![Class](https://img.shields.io/badge/Class-Blueprint-8B0000?style=flat-square)
![Object](https://img.shields.io/badge/Object-Instance-800000?style=flat-square)
![Constructor](https://img.shields.io/badge/Constructor-Initialization-darkred?style=flat-square)

</details>

---

<details>
<summary><b>🔒 03 — Encapsulation</b></summary>
<br/>

The OOP principle of **hiding internal data** and controlling access through methods. Covers getter and setter methods, access modifiers, and why encapsulation leads to safer and more maintainable code.

![Private](https://img.shields.io/badge/private-Hidden-8B0000?style=flat-square)
![Getter](https://img.shields.io/badge/Getter-Read%20Access-800000?style=flat-square)
![Setter](https://img.shields.io/badge/Setter-Write%20Access-darkred?style=flat-square)

</details>

---

<details>
<summary><b>🧬 04 — Inheritance</b></summary>
<br/>

Modeling **IS-A relationships** and reusing code from parent classes. Covers `extends`, `super`, single/multilevel/hierarchical inheritance, method overriding, and the `@Override` annotation.

![extends](https://img.shields.io/badge/extends-Inheritance-8B0000?style=flat-square)
![super](https://img.shields.io/badge/super-Parent%20Access-800000?style=flat-square)
![Override](https://img.shields.io/badge/@Override-Method%20Override-darkred?style=flat-square)

</details>

---

<details>
<summary><b>🔀 05 — Polymorphism</b></summary>
<br/>

Designing **flexible code** where one interface serves many forms. Covers compile-time polymorphism (method overloading), runtime polymorphism (method overriding), upcasting, and dynamic dispatch.

![Overloading](https://img.shields.io/badge/Overloading-Compile--time-8B0000?style=flat-square)
![Overriding](https://img.shields.io/badge/Overriding-Runtime-800000?style=flat-square)
![Dispatch](https://img.shields.io/badge/Dynamic-Dispatch-darkred?style=flat-square)

</details>

---

<details>
<summary><b>🎭 06 — Abstract Class</b></summary>
<br/>

Defining a **partial blueprint** that cannot be instantiated directly. Covers the `abstract` keyword, abstract methods (no body), concrete methods (shared implementation), and constructors in abstract classes.

![abstract](https://img.shields.io/badge/abstract-Partial%20Class-8B0000?style=flat-square)
![No Instance](https://img.shields.io/badge/Cannot-Instantiate-800000?style=flat-square)
![Override](https://img.shields.io/badge/Must-Override-darkred?style=flat-square)

</details>

---

<details>
<summary><b>📋 07 — Interfaces</b></summary>
<br/>

Defining **contracts** that classes agree to fulfill. Covers `interface`, `implements`, multiple interface implementation, functional interfaces, and a full comparison between interfaces and abstract classes.

![interface](https://img.shields.io/badge/interface-Contract-8B0000?style=flat-square)
![implements](https://img.shields.io/badge/implements-Fulfill-800000?style=flat-square)
![Multiple](https://img.shields.io/badge/Multiple-Inheritance-darkred?style=flat-square)

</details>

---

<details>
<summary><b>🔲 08 — Inner Classes</b></summary>
<br/>

Classes defined **inside other classes** for better encapsulation. Covers all four types with their differences, access rules, and use cases.

| Type | Description |
|---|---|
| Member Inner Class | Non-static — accesses outer instance |
| Static Nested Class | Static — no outer instance needed |
| Local Inner Class | Defined inside a method |
| Anonymous Inner Class | Unnamed, defined inline |

</details>

---

<details>
<summary><b>⚙️ 09 — Static and Final</b></summary>
<br/>

**`static`** — members that belong to the class, not any object.
Covers static variables (shared), static methods (cannot be overridden), static blocks, and static nested classes.

**`final`** — members that cannot be changed.
Covers final variables, final methods, final classes, final parameters, and the `public static final` constant pattern.

![static](https://img.shields.io/badge/static-Class%20Level-8B0000?style=flat-square)
![final](https://img.shields.io/badge/final-Immutable-800000?style=flat-square)
![Constant](https://img.shields.io/badge/public%20static%20final-Constant-darkred?style=flat-square)

</details>

---

<details>
<summary><b>📦 10 — Packages</b></summary>
<br/>

Organizing Java classes into **logical namespaces**. Covers `package` declaration, `import` statements (specific and wildcard), static imports, built-in Java packages, multi-package project structure, and naming conventions.

![package](https://img.shields.io/badge/package-Namespace-8B0000?style=flat-square)
![import](https://img.shields.io/badge/import-Access-800000?style=flat-square)
![Convention](https://img.shields.io/badge/com.iug.java3-Naming-darkred?style=flat-square)

</details>

---

<details>
<summary><b>⚠️ 11 — Exception Handling</b></summary>
<br/>

Writing programs that **detect and recover from runtime errors** gracefully. Covers the exception hierarchy, checked vs unchecked exceptions, `try/catch/finally`, `throw`, `throws`, multi-catch, try-with-resources, and custom exception classes.

![try](https://img.shields.io/badge/try-catch-8B0000?style=flat-square)
![throw](https://img.shields.io/badge/throw-throws-800000?style=flat-square)
![Custom](https://img.shields.io/badge/Custom-Exception-darkred?style=flat-square)

</details>

---

<details>
<summary><b>📂 12 — Java IO Streams</b></summary>
<br/>

Reading and writing data between a program and files.

**📄 Text Files (Character Streams)**

| Class | Role |
|---|---|
| `Scanner` | Reads and parses text from console or file |
| `PrintWriter` | Writes formatted text to console or file |

**🗄️ Binary Files (Byte Streams)**

| Class | Role |
|---|---|
| `FileInputStream` / `FileOutputStream` | Read and write raw bytes |
| `DataInputStream` / `DataOutputStream` | Read and write Java primitives in binary format |

</details>

---

<details>
<summary><b>🔧 13 — Java Generics</b></summary>
<br/>

Writing **reusable, type-safe code** that works with any data type. Covers generic classes, multiple type parameters, generic methods, bounded type parameters, and wildcards.

![Generic](https://img.shields.io/badge/%3CT%3E-Generic%20Class-8B0000?style=flat-square)
![Bounded](https://img.shields.io/badge/T%20extends%20Number-Bounded-800000?style=flat-square)
![Wildcard](https://img.shields.io/badge/%3F%3E-Wildcard-darkred?style=flat-square)

</details>
